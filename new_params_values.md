# Proposed Default Explainer Parameter Changes

## Goal

Improve the default quality and stability of explanations generated by
`mprov3_explainer/scripts/run_explanations.py`, with special attention to
molecular graphs and binding-affinity use cases where diffuse or degenerate
masks are a common failure mode.

This document is a plan only. No code defaults should be changed until this
plan is reviewed and approved.

## Why These Parameters

The most consequential defaults in the current pipeline are the ones that
change the explainer optimization budget or attribution stability, not the
reporting-only metrics.

Highest-impact candidates:

1. `DEFAULT_PG_EXPLAINER_EPOCHS`
2. `DEFAULT_GNN_EXPLAINER_EPOCHS`
3. `DEFAULT_IG_N_STEPS`
4. `DEFAULT_PGM_NUM_SAMPLES`

Secondary candidate:

5. `DEFAULT_PG_EXPLAINER_LR`

Not proposed as a default change for now:

1. `top_k_fraction`
2. mask-spread filtering
3. seed
4. split or fold-selection behavior

Reason: those mostly affect evaluation protocol, comparability, or diagnostics
more than the intrinsic quality of the produced explanations.

## Current Defaults

From `mprov3_gine_explainer_defaults/mprov3_gine_explainer_defaults/explainer_algorithm_defaults.py`:

- `DEFAULT_GNN_EXPLAINER_EPOCHS = 200`
- `DEFAULT_GNN_EXPLAINER_LR = 0.01`
- `DEFAULT_PG_EXPLAINER_EPOCHS = 30`
- `DEFAULT_PG_EXPLAINER_LR = 0.003`
- `DEFAULT_IG_N_STEPS = 32`
- `DEFAULT_PGM_NUM_SAMPLES = 100`

## Proposed New Defaults

### 1. PGExplainer training epochs

- Current: `30`
- Proposed: `100`

Reason:

- PGExplainer is the most likely method here to underfit its explanation MLP.
- The current pipeline already tracks degenerate masks, and PGExplainer is one
  of the methods most likely to benefit from a larger training budget.
- On graph-level chemistry tasks, 30 epochs is often too small for stable
  edge-mask learning.

Expected tradeoff:

- Higher runtime
- Better mask stability
- Lower chance of near-constant masks

### 2. GNNExplainer optimization epochs

- Current: `200`
- Proposed: `400`

Reason:

- GNNExplainer is an instance-wise optimization method; explanation quality can
  improve noticeably with a longer mask optimization schedule.
- Molecular graphs are not huge, so doubling the optimization budget is a
  reasonable default shift.

Expected tradeoff:

- Higher runtime per explained graph
- Potentially sharper and less noisy masks

### 3. Integrated Gradients steps

- Current: `32`
- Proposed: `128`

Reason:

- IG quality is sensitive to integration granularity.
- A higher step count is usually a safe default when the goal is explanation
  quality rather than speed.
- This is especially useful when the learned decision surface is irregular
  around chemically meaningful substructures.

Expected tradeoff:

- Higher runtime
- Smoother, more stable attributions

### 4. PGMExplainer samples

- Current: `100`
- Proposed: `300`

Reason:

- PGM-style perturbation methods can be noisy at low sample counts.
- More samples should improve stability of importance estimates across graphs.

Expected tradeoff:

- Higher runtime
- Lower sampling variance

### 5. PGExplainer learning rate

- Current: `0.003`
- Proposed: keep unchanged for the first revision

Reason:

- The epoch budget is the cleaner first intervention.
- Changing both epoch count and learning rate at once makes it harder to
  attribute improvements or regressions.
- `0.003` is already a conservative value.

## Validation Plan After Code Change

After updating defaults, compare old vs new runs using the same seed and fold(s).

Recommended checks:

1. Run explanations on the validation split, not only the test split.
2. Compare `num_valid` and `num_degenerate_mask` per explainer.
3. Compare `mean_paper_f1_fidelity` and `mean_pyg_characterization`.
4. Inspect a small visual sample of explanations for medicinal-chemistry
   plausibility.
5. Watch runtime growth, especially for IG and GNNExplainer.

## Success Criteria

The default change is worth keeping if it produces one or more of the
following without unacceptable runtime cost:

1. fewer degenerate masks
2. more valid explanations
3. better fidelity-based aggregate metrics
4. more localized and chemically plausible highlighted motifs

## Risks

1. Runtime may increase substantially, especially for `GNNEXPL` and IG.
2. Higher optimization budgets can occasionally make masks sharper but less
   stable across seeds.
3. Better benchmark metrics do not automatically mean better scientific
   interpretability, so visual inspection should remain part of evaluation.
